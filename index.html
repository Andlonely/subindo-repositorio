Criando um repositório no GitHub

Ok, vamos começar do começo.

Você já criou um projeto local para enviar para o GitHub?

Certo, então primeiro você precisa criar um projeto local.

Depois de criar um projeto local, volte aqui e clique em "Ok".

Ótimo! Agora que você tem um projeto local, o próximo passo é inicializar o Git.

Você já iniciou o Git?

Ok, então vamos inicializar o Git.

Abra o terminal ou Git Bash e Navegue até a pasta onde está o diretório do seu projeto usando o comando cd. Por exemplo, se o seu projeto estiver na pasta

"Documents/MeuProjeto"

Você pode navegar até ela digitando

cd Documents/MeuProjeto..

Em seguida, execute o seguinte comando:

git init

Depois de iniciar o Git, volte aqui e clique em "Ok".

Excelente! Agora que o Git está inicializado, o próximo passo é criar um arquivo README.md para o seu projeto.

Você já criou um arquivo README.md?

Ok, então vamos criar o arquivo README.md.

Abra o terminal ou Git Bash e navegue até o diretório raiz do seu projeto.

Em seguida, execute o seguinte comando:

touch README.md

Depois de criar o arquivo README.md, volte aqui e clique em "Ok".

Ótimo! Agora que você tem um arquivo README.md, o próximo passo é adicionar o arquivo README.md ao Git.

Você já adicionou o arquivo README.md ao Git?

Ok, então vamos adicionar o arquivo README.md ao Git.

No terminal ou Git Bash, execute o seguinte comando:

git add README.md

Depois de adicionar o arquivo README.md, volte aqui e clique em "Ok".

Ótimo! Agora que o arquivo README.md foi adicionado ao Git, o próximo passo é fazer um commit das mudanças.

Você já fez um commit das mudanças?

Ok, então vamos fazer um commit das mudanças.

No terminal ou Git Bash, execute o seguinte comando:

git commit -m "Adicionado arquivo README.md"

Depois de fazer o commit das mudanças, volte aqui e clique em "Ok".

Perfeito! Agora que o commit foi feito, o próximo passo é criar um repositório no GitHub.

Você já criou um repositório no GitHub?

Ok, então vamos criar um repositório no GitHub.

Abra o GitHub e faça login na sua conta.

Em seguida, clique no botão verde "New" na página inicial do GitHub.

Dê um nome ao seu repositório e, em seguida, clique no botão "Create repository".

Depois de criar o repositório, volte aqui e clique em "Ok".

Ótimo! Agora que o repositório foi criado no GitHub, o próximo passo é conectar o repositório local ao repositório remoto.

Você sabe qual é o URL do repositório remoto?

Ok, então vamos obter o URL do repositório remoto.

No GitHub, vá até o seu repositório e clique no botão "Clone or download".

Copie o URL do repositório.

Depois de obter o URL do repositório, volte aqui e clique em "Ok".

Excelente! Agora que temos o URL do repositório remoto, o próximo passo é conectar o repositório local ao repositório remoto.

No terminal ou Git Bash, execute o seguinte comando:

git remote add origin URL_DO_REPOSITORIO_REMOTO

Lembre-se de substituir "URL_DO_REPOSITORIO_REMOTO" pelo URL que você copiou anteriormente.

Depois de conectar o repositório local ao repositório remoto, volte aqui e clique em "Ok".

Maravilhoso! Agora que o repositório local está conectado ao repositório remoto, o próximo passo é enviar as mudanças para o repositório remoto.

Você já enviou as mudanças para o repositório remoto?

Ok, então vamos enviar as mudanças para o repositório remoto.

No terminal ou Git Bash, execute o seguinte comando:

git push -u origin master

Depois de enviar as mudanças para o repositório remoto, volte aqui e clique em "Ok".

Parabéns! Você criou um repositório no GitHub e adicionou um arquivo HTML com instruções sobre como criar um repositório no Git.
